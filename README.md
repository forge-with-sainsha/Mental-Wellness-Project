
<h1>🌟 Student Stress Monitoring & Guidance System</h1> 

 <p> A secure, full-stack web application designed for educational institutions to monitor student stress levels and deliver automated, personalized
  wellness support. It includes a student self-assessment portal and a centralized admin monitoring panel.
</p>

 <h3> 🚀 Key Features</h3>

 - 💻 Secure Accounts: Safe registration and login secured with password hashing and session tokens
- Interactive Assessments: Standardized stress-evaluation quizzes with automated, instant scoring
- Progress Dashboard: A visual history tracking past results and wellness progress
- Progress Dashboard: A visual history tracking past results and wellness progress
- Personalized Guidance: Automated action plans and wellness resources customized to each student's stress score

   <h3> 🛡️ Admin Dashboard</h3>

 - Campus Overview: Real-time metrics showing total registrations and average student stress levels
- Student Registry: Comprehensive list to monitor student history and flag those needing high-priority support
- Dynamic Control: Administrative controls to update assessment questions and manage wellness recommendations instantly


   <h3> 🛠️ Technology Stack & Architecture</h3>
   
 - Backend Server: Built with Node.js and Express.js to handle secure API requests.
 - Database: Structured MySQL relational database with foreign key associations.
 -  Frontend UI: Built with responsive HTML5, CSS3, and Vanilla JavaScript (no heavy frameworks needed, showcasing strong core web-dev skills).

  <h3>🔒 Security Best Practices Implemented</h3>  

<p>This project is built following production-level security standards:</p>

 - Password Protection: Passwords are encrypted using industry-standard hashing algorithms before saving.
 -  Role-Based Security: Strict access controls prevent student accounts from accessing admin APIs.
 - Input Validation: Robust server-side validation checks all inputs to defend against common web attacks.
 -  Rate Limiting: Protects authenticating endpoints (login/register) from brute-force login attempts.
